# 抖音群聊续火花使用说明

## 功能说明

本项目现已支持**抖音群聊续火花**功能，可以自动向指定的抖音群聊发送续火花消息。

## 配置方法

### 1. 环境变量配置

在 `TASKS` 环境变量中，为每个账号添加 `group_targets` 字段，指定要发送消息的群聊名称列表。

#### 配置示例

```json
[
  {
    "username": "我的抖音账号",
    "unique_id": "user123",
    "targets": ["好友昵称1", "好友昵称2"],
    "group_targets": ["闺蜜群", "家人群", "工作群"]
  }
]
```

### 2. 字段说明

- `username`: 账号备注名（用于日志显示）
- `unique_id`: 账号唯一标识（用于关联 Cookie）
- `targets`: 私聊好友列表（昵称或抖音号）
- `group_targets`: **群聊名称列表**（新增字段）

### 3. Cookie 配置

与私聊续火花相同，需要配置对应账号的 Cookie：

```bash
COOKIES_USER123=你的Cookie内容
```

## 使用场景

### 场景 1：只发送群聊消息

```json
[
  {
    "username": "我的账号",
    "unique_id": "user123",
    "targets": [],
    "group_targets": ["闺蜜群", "家人群"]
  }
]
```

### 场景 2：同时发送私聊和群聊

```json
[
  {
    "username": "我的账号",
    "unique_id": "user123",
    "targets": ["好友A", "好友B"],
    "group_targets": ["闺蜜群", "家人群"]
  }
]
```

### 场景 3：只发送私聊消息（原有功能）

```json
[
  {
    "username": "我的账号",
    "unique_id": "user123",
    "targets": ["好友A", "好友B"]
  }
]
```

## 执行流程

1. 程序会先处理 `targets` 中的私聊好友
2. 然后处理 `group_targets` 中的群聊
3. 每个聊天发送完消息后会等待 2 秒再继续

## 注意事项

1. **群聊名称必须完全匹配**：请确保 `group_targets` 中的群聊名称与抖音中显示的完全一致
2. **群聊查找顺序**：程序会从上到下滚动群聊列表查找目标群聊
3. **未找到群聊**：如果某个群聊未找到，会在日志中显示警告信息
4. **消息内容**：群聊和私聊使用相同的消息模板（`MESSAGE_TEMPLATE` 环境变量）

## 常见问题

### Q1: 群聊名称找不到怎么办？

A: 请检查以下几点：
- 群聊名称是否完全匹配（包括空格、特殊字符）
- 该群聊是否在消息列表中（可能需要先手动发送一条消息激活）
- 检查日志中是否有 "未找到群聊" 的警告信息

### Q2: 可以同时给多个群聊发消息吗？

A: 可以，在 `group_targets` 数组中添加多个群聊名称即可。

### Q3: 群聊消息和私聊消息内容一样吗？

A: 是的，目前使用相同的消息模板。如果需要区分，可以后续扩展配置。

## 技术实现

- 群聊列表与私聊列表在同一页面，通过标签页切换
- 使用 XPath 定位群聊标签页和群聊列表项
- 支持滚动加载更多群聊
- 自动检测列表底部，避免无限滚动
